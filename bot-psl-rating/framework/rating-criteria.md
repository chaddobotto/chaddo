# Bot PSL Rating Criteria

**Detailed Scoring Rubrics for Each Dimension**

---

## Overview

This document provides comprehensive scoring guidelines for evaluating bots across all five PSL dimensions. Each dimension is scored 0-10, with specific criteria for each score level.

---

## 1. Architectural Symmetry (AS)

**Weight**: 25% of total PSL

### Scoring Rubric

#### 10 — Perfect Symmetry (0.1%)
- Every package has a mirror counterpart
- 100% consistent naming conventions
- Zero API inconsistencies
- Perfect directory hierarchy balance
- Error handling is completely uniform
- Every module follows identical patterns

**Example Bots**: None exist in practice

#### 9 — Near Perfect (1%)
- 95%+ package symmetry
- One minor naming inconsistency
- All APIs follow nearly identical patterns
- Negligible asymmetry
- Professional-grade organization

**Example**: Theoretical "Platonic Bot" implementation

#### 8 — Excellent (5%)
- 90%+ package symmetry
- Very consistent naming
- Minor API pattern variations
- Clear organizational structure
- Well-maintained symmetry

**Example**: High-quality enterprise frameworks

#### 7 — Very Good (10%)
- 85%+ package symmetry
- Mostly consistent naming
- Some API pattern variations
- Good structure with few exceptions
- Above-average organization

#### 6 — Above Average (20%)
- 75%+ package symmetry
- Generally consistent naming
- Noticeable API variations
- Decent structure
- Some organizational issues

#### 5 — Average (38%)
- 50-75% package symmetry
- Inconsistent naming in places
- Multiple API pattern styles
- Basic structure
- Mixed organization quality

**Example**: Typical mid-sized open source project

#### 4 — Below Average (15%)
- 30-50% package symmetry
- Frequent naming inconsistencies
- Poor API consistency
- Weak structure
- Organizational problems

#### 3 — Poor (8%)
- 10-30% package symmetry
- Inconsistent naming throughout
- Chaotic API patterns
- Little structure
- Significant asymmetry

#### 2 — Very Poor (2%)
- <10% package symmetry
- No naming conventions
- Random API patterns
- No discernible structure
- Severe asymmetry

#### 1 — Terminal (0.5%)
- Zero symmetry
- Complete chaos
- No patterns whatsoever
- Unorganized mess

#### 0 — Non-existent (0.4%)
- No code structure exists
- Broken/deleted repository

## Composite PSL Calculation

### Formula

```
PSL = (AS × 0.25) + (FP × 0.25) + (HC × 0.20) + (MP × 0.15) + (SP × 0.15)
```

### Example Calculation

For a bot with scores:
- AS: 7.5
- FP: 6.8
- HC: 8.2
- MP: 5.5
- SP: 7.0

```
PSL = (7.5 × 0.25) + (6.8 × 0.25) + (8.2 × 0.20) + (5.5 × 0.15) + (7.0 × 0.15)
PSL = 1.875 + 1.700 + 1.640 + 0.825 + 1.050
PSL = 7.09
```

**Final PSL**: 7.1 (rounded to one decimal)

## Inter-Rater Agreement Guidelines

When rating bots, achieve consistency by:

1. **Calibration**: Rate 10 reference bots first
2. **Compare**: Check scores against benchmark bots
3. **Justify**: Document reasoning for each score
4. **Review**: Have second rater verify scores
5. **Adjust**: Reconcile differences through discussion

**Target Agreement**: >80% within 0.5 PSL points

---

## Rating Best Practices

### Do's
✅ Rate multiple times and average
✅ Compare against similar bots
✅ Document specific evidence
✅ Consider all dimensions equally (by weight)
✅ Use decimal precision

### Don'ts
❌ Let personal preference bias scores
❌ Compare across different bot categories
❌ Rush ratings without analysis
❌ Ignore objective metrics
❌ Round scores prematurely
