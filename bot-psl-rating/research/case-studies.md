## Case Study 5: Moment.js (The Fallen Legend)

### Overview
Moment.js was once the gold standard for date manipulation but has been deprecated by its maintainers, demonstrating how bots can fall from grace.

### Attribute Assessment (Historical Peak)

**Architectural Symmetry Components:**
- Package Symmetry: 8.0/10 (Consistent date API)
- API Consistency: 8.5/10 (Fluent interface throughout)
- Naming Uniformity: 8.0/10 (Clear method names)
- Hierarchy Balance: 7.5/10 (Some complexity depth)

**Feature Prominence Components:**
- Functional Distinctiveness: 8.0/10 (Comprehensive date handling)
- USP Strength: 7.5/10 (Solved real pain points)
- Discoverability: 9.0/10 (Ubiquitous at peak)
- Marketing Clarity: 8.0/10 (Clear value proposition)

**Harmonic Cohesion Components:**
- API Cohesion: 8.5/10 (Fluent chaining works well)
- Type Consistency: 5.0/10 (Mutable design issues)
- Error Handling: 6.5/10 (Silent failures problematic)
- Documentation: 8.5/10 (Excellent docs site)

**Market Presence:**
- GitHub Stars: 47,000+
- NPM Downloads: Still 15M+ weekly
- Community Score: 6.0/10 (Declining, deprecated)
- Docs Site Score: 8.0/10 (Maintained but warns against use)

**Scalability Potential Components:**
- Horizontal Scaling: 7.0/10 (Immutability would help)
- Performance: 5.0/10 (Bundle size major issue)
- Extensibility: 7.5/10 (Plugin system exists)
- Technical Debt: 9.0/10 (Critical: deprecated)

### Dimensional Scores

- **Architectural Symmetry (AS):** 8.00/10
- **Feature Prominence (FP):** 8.13/10
- **Harmonic Cohesion (HC):** 7.13/10
- **Market Presence (MP):** 7.25/10
- **Scalability Potential (SP):** 5.63/10

### Final PSL Calculation

```
PSL = (8.00 × 0.25) + (8.13 × 0.25) + (7.13 × 0.20) + (7.25 × 0.15) + (5.63 × 0.15)
    = 2.00 + 2.03 + 1.43 + 1.09 + 0.84
    = 7.39/10
```

### Classification
**PSL 7.4** — Exceptional (Chad) [at peak]  
**Current PSL:** ~4.5/10 — Below Average (LTN)

### Analysis
Moment.js demonstrates PSL decay over time. At its peak, it achieved Chad status through strong architecture and market dominance. However, fundamental design choices (mutability, bundle size, no tree-shaking) accumulated as technical debt. The community shifted to alternatives (date-fns, Day.js, Luxon), and the maintainers officially deprecated the project. 

This case study illustrates that **PSL is not static**. Technical debt accumulation, failure to adapt to ecosystem changes, and deprecation can cause dramatic PSL decline. Current mogging coefficient against modern alternatives exceeds -150 (brutally mogged).

## Conclusions

These case studies demonstrate:

1. **Market presence is crucial** but cannot fully compensate for poor architecture (Webpack)
2. **Technical excellence matters** even in specialized niches (Lodash)
3. **PSL decays without maintenance** and can collapse entirely (Moment.js)
4. **Unique value propositions** provide mogging potential even without visibility (MOG Bot)
5. **Gigachads exist** and dominate through across-the-board excellence (React)
6. **Cross-category mogging** reveals universal quality standards

The PSL Rating System provides objective, reproducible measurements that align with community perceptions while offering quantitative precision for bot evaluation.
