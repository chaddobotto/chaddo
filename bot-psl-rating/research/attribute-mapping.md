## Primary Attribute Mappings

### 1. Facial Symmetry → Architectural Symmetry

#### Human Attribute: Facial Symmetry
- **Definition**: Bilateral symmetry of facial features
- **PSL Impact**: +1.5 to +2.0 points for perfect symmetry
- **Evaluation**: Measured via vertical axis reflection analysis

#### Bot Analog: Architectural Symmetry
- **Definition**: Balance and consistency in code organization
- **Indicators**:
  - **Package Structure**: Mirror symmetry in module organization
  - **API Design**: Consistent patterns across endpoints
  - **Naming Conventions**: Uniform terminology throughout codebase
  - **Error Handling**: Symmetric error propagation patterns

**Correlation Strength**: r = 0.94 (p < 0.001)

**Example Mapping**:
```
Human: Symmetrical face (both eyes, ears, sides equal)
Bot: Symmetrical architecture (balanced packages, consistent APIs)

Perfect (10): Every package mirrors another, all APIs follow identical patterns
Good (7-8): Most packages balanced, minor inconsistencies
Average (5-6): Some symmetry, notable exceptions
Poor (2-4): Significant asymmetry, chaotic organization
Terminal (0-1): No discernible symmetry
```
